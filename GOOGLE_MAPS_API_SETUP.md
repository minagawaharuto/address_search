## 1. Google Maps APIキーの取得

### ステップ1: Google Cloud Consoleにアクセス

1. https://console.cloud.google.com/ にアクセス
2. Googleアカウントでログイン
3. 新しいプロジェクトを作成（または既存のプロジェクトを選択）

### ステップ2: プロジェクトの作成

1. 画面上部の「プロジェクトを選択」をクリック
2. 「新しいプロジェクト」をクリック
3. プロジェクト名を入力（例: `address-search-system`）
4. 「作成」をクリック

### ステップ3: 請求先アカウントの設定

**注意**: APIキーを使用するには、クレジットカードの登録が必要です。
ただし、**無料枠内（月28,000リクエスト）なら課金されません**。

1. 左側メニューから「お支払い」を選択
2. 「請求先アカウントをリンク」をクリック
3. クレジットカード情報を入力
4. 無料トライアル（$300分のクレジット）が適用されます

### ステップ4: Places APIの有効化

1. 左側メニューから「APIとサービス」→「ライブラリ」を選択
2. 検索ボックスに「Places API」と入力
3. **「Places API」** をクリック
4. 「有効にする」をクリック

### ステップ5: APIキーの作成

1. 左側メニューから「APIとサービス」→「認証情報」を選択
2. 上部の「認証情報を作成」→「APIキー」をクリック
3. APIキーが生成されます
4. **このキーをコピーして安全に保管してください**

### ステップ6: APIキーの制限設定（推奨）

セキュリティのため、APIキーに制限を設定することを推奨します：

1. 生成されたAPIキーの右側の「編集」アイコンをクリック
2. 「アプリケーションの制限」で以下を選択：
   - 開発中: 「なし」
   - 本番環境: 「IPアドレス」を選択し、使用するIPアドレスを入力
3. 「APIの制限」で「キーを制限」を選択
4. 「Places API」にチェックを入れる
5. 「保存」をクリック

## 2. 料金について

### 無料枠

Google Maps Platform には無料枠があります：

- **月額$200のクレジット**（毎月自動付与）
- **Places API (Text Search) の料金**: 1,000リクエストあたり $32

計算：
```
$200 ÷ $32 × 1,000 = 約6,250リクエスト/月 無料
```

実際には、複数のAPIを組み合わせて使用するため、**月28,000リクエスト程度まで無料**で利用可能です。

### 使用量の確認方法

1. Google Cloud Console にアクセス
2. 左側メニュー「APIとサービス」→「ダッシュボード」
3. Places APIの使用量グラフで確認

### 予算アラートの設定（推奨）

意図しない課金を防ぐため、予算アラートを設定することを推奨します：

1. 左側メニュー「お支払い」→「予算とアラート」
2. 「予算を作成」をクリック
3. 予算額を設定（例: $10）
4. アラートのしきい値を設定（例: 50%, 90%, 100%）
5. メール通知を有効化

## 3. APIキーの設定

### 方法1: config.txtファイルを作成（推奨）

1. プロジェクトフォルダ（`住所検索システム`）に `config.txt` ファイルを作成
2. 以下の内容を記載：

```
GOOGLE_MAPS_API_KEY=ここにあなたのAPIキーを貼り付け
```

例：
```
GOOGLE_MAPS_API_KEY=AIzaSyDXXXXXXXXXXXXXXXXXXXXXXXXXXXX
```

**注意**:
- `=` の前後にスペースを入れないでください
- APIキーをそのまま貼り付けてください
- `config.txt` は `.gitignore` に追加して、Gitにコミットしないでください

### 方法2: 環境変数に設定

Windows の場合：

```cmd
setx GOOGLE_MAPS_API_KEY "あなたのAPIキー"
```

その後、コマンドプロンプトを再起動してください。

## 5. トラブルシューティング

### エラー: API Key 未設定

**症状**: 画面に「⚠ API Key 未設定」と表示される

**解決方法**:
1. `config.txt` ファイルが正しい場所にあるか確認
2. ファイルの内容を確認：
   ```
   GOOGLE_MAPS_API_KEY=あなたのAPIキー
   ```
3. APIキーに余分なスペースや改行がないか確認
4. アプリケーションを再起動

### エラー: REQUEST_DENIED

**症状**: 住所検索時に「APIエラー: REQUEST_DENIED」と表示される

**原因**: Places APIが有効化されていない

**解決方法**:
1. Google Cloud Console にアクセス
2. 「APIとサービス」→「ライブラリ」
3. 「Places API」を検索して有効化

### エラー: OVER_QUERY_LIMIT

**症状**: 「APIエラー: OVER_QUERY_LIMIT」と表示される

**原因**: 無料枠を超えた、またはレート制限

**解決方法**:
1. 使用量を確認（Google Cloud Console → APIダッシュボード）
2. しばらく待ってから再試行
3. 必要に応じて有料プランを検討

### エラー: INVALID_REQUEST

**症状**: 特定の店舗で「住所が見つかりませんでした」と表示される

**原因**: 検索クエリが不適切、または店舗情報が登録されていない

**解決方法**:
1. 会社名のスペルを確認
2. 支店名の表記を確認（例: 「東京店」→「東京支店」）
3. 手動でGoogle Mapsで検索して確認

## 📊 使用例

### 良い例

**会社名**: ローソン

**結果**: 〒663-8142 兵庫県西宮市鳴尾浜2丁目1-31

### 注意が必要な例

**会社名**: ABC株式会社
**支店名**: 本社

→ 一般的な名前すぎて、正確な結果が得られない可能性があります。
より具体的な情報（「ABC株式会社 東京本社」など）を入力してください。

## 🔒 セキュリティのベストプラクティス

1. **APIキーを公開しない**
   - GitHubなどにコミットしない
   - `.gitignore` に `config.txt` を追加

2. **APIキーに制限を設定**
   - IPアドレス制限
   - API制限（Places APIのみ）
   - リファラー制限（Webアプリの場合）

3. **使用量を監視**
   - 予算アラートを設定
   - 定期的に使用量を確認

4. **APIキーの定期的なローテーション**
   - セキュリティのため、3〜6ヶ月ごとにAPIキーを再生成

## 💡 ヒント

### より正確な検索のために

1. **会社名は正式名称を使用**
   - 良い例: ローソン、セブン-イレブン
   - 悪い例: LAWSON、7-11

2. **支店名は一般的な表記を使用**
   - 店舗、支店、営業所など

3. **地域名を含める**
   - 「東京店」より「東京渋谷店」の方が精度が高い

### コスト削減のために

1. **バッチ処理**
   - 大量のデータは一度にまとめて処理

2. **キャッシュの活用**
   - 同じ店舗は再検索しない（将来の機能として実装予定）

3. **無料枠の監視**
   - 毎月の使用量を確認
